                    📖 Atlas System Guide & Documentation
                
          
                
            
          
            
            
          
                
          
                    Welcome to the **Atlas Documentation Pipeline**. This platform automatically ingests unstructured raw files, extracts key architectural, financial, and timeline facts, resolves internal conflicts, and synthesizes unified technical specifications.
                
          

                
          
                    
                    
          
                        
          📥 Drag & Drop Workspace Loader
          
                        
          Upload local files instantly to dynamically load them into the system memory:
          
                        
            
                            
          • Drag raw files onto the dotted workspace target or click the card to browse.
            • 
                            
          • Supports plain text (.txt), markdown (.md), JSON configurations (.json), and code files (.py, .java).
            • 
                            
          • Loaded files can be edited in real-time or deleted prior to executing extraction.
            • 
                        
          
                    
          

                    
                    
          
                        
          ⚡ Smart MD5 Fact Caching
          
                        
          Avoid redundant processing and expensive API consumption with the built-in cache:
          
                        
            
                            
          • The pipeline computes cryptographic MD5 hashes of file contents upon ingestion.
            • 
                            
          • If a file hasn't changed, facts are instantly retrieved from .fact_cache.json.
            • 
                            
          • This cuts down external LLM calls by up to **80%** on recurring syntheses.
            • 
                        
          
                    
          

                    
                    
          
                        
          💾 Persistent Config Defaults
          
                        
          Configure settings once and preserve them across your working sessions:
          
                        
            
                            
          • Select your LLM service provider and model identifier, and enter your authorization token.
            • 
                            
          • Click **"Save Default"** to write parameters to the server's config.json.
            • 
                            
          • Future page loads will automatically populate credentials, keeping the API seamless.
            • 
                        
          
                    
          

                    
                    
          
                        
          💻 CLI & Automated Batch Scripts
          
                        
          Run the pipeline from standard Windows terminals or using automated double-click batch files:
          
                        
            
                            
          • Run Pipeline: Use python run_cli.py or run run_cli_live.bat.
            • 
                            
          • Offline Simulation: Execute run_cli_simulation.bat to test the pipeline flow for free.
            • 
                            
          • Start Dashboard Server: Double-click start_server.bat to launch the local web server.
            • 
                        
          
                    
          
                
          

                
          
                    
          📄 Reference Template Documentation
          
                    
          
                        To assist in designing conforming materials, the system has a pre-formatted **documentation.docx** reference standard stored at the workspace root. This template showcases sample layout formatting, structures, and conflict handling metrics for engineering review.
